# User Stories

## Overview
This document outlines user stories for the Highland Wildlife Survey Data Pipeline.

## User Story 1: Submit Data
As a volunteer, I want to submit wildlife sightings using CSV files, so that my observations can be processed by the system.

## User Story 2: Validate Data
As a user of the system, I want the system to validate records and store correct data, so that the data is accurate and reliable.

## User Story 3: Store Survey Data
As a user of the system, I want the data to be stored in a database, so that it can be accessed and analysed later.

## User Story 4: Generate Reports
As a partner organisation, I want the system to generate reports, so that the data can be shared and used externally.
